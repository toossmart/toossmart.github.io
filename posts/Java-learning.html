DTO、Entity、VO

​ DTO在controller层接收前端传出的数据，在service层封装成Entity。如果是对数据库的增删改，那将Entity传入mapper层，执行INSERT/UPDATE，后在controller层返回Result.success()，如果是对数据库的查询操作，那将Entity转化为VO：作用：数据脱敏（去掉密码）、格式化（日期转字符串）、组合数据（计算总价）。在Controller层将VO封装到Result<T> 对象中，返回给前端。

列表复制操作

List<ShoppingCart> shoppingCartList = orderDetailList.stream().map(x -> {
    ShoppingCart shoppingCart = new ShoppingCart();
    BeanUtils.copyProperties(x, shoppingCart, "id");
    shoppingCart.setUserId(userId);
    shoppingCart.setCreateTime(LocalDateTime.now());
    return shoppingCart;
}).collect(Collectors.toList())
  • 将orderDetailList中的元素复制到shoppingCartList，并且两个列表中的元素并不相同
  • .stream()：将List转换为流
  • .map(x -> { ... }): 遍历列表中的每一个元素（用 x 代表每一个 OrderDetail 对象），并将其转换为一个新的对象（ShoppingCart）
  • .collect(Collectors.toList()): 将处理完的流重新收集成一个新的 List<ShoppingCart> 集合
  • BeanUtils.copyProperties(x, shoppingCart, "id");：忽略列表，在复制属性时，不复制 id 字段。
